How It Works

Agents, Skills, and Tools

Each demo is one agent. Each agent has skills. Skills connect to tools and data directly, not through a web interface.

🤖
Agent

An autonomous workflow that makes decisions, calls skills, and delivers a result. Each demo builds one agent.

Example: Transcript Analysis Agent
⚡
Skill

A reusable step an agent can call. Like a recipe: specific instructions for one task. Multiple agents share the same skills.

Example: /transcript-downloader, /ai-digester
🔌
Tool (MCP)

A direct connection to data or a service. Claude talks to APIs, databases, and files directly, not through a browser or web UI.

Example: FMP API, Google Drive, Slack
🔗
Direct data access, not web scraping

Claude connects directly to APIs and databases via MCP (Model Context Protocol). No browser automation, no copy-paste. The agent reads Bloomberg, FactSet, or your internal data the same way a developer would: through a structured API call.

The skill calls the Financial Modeling Prep API with the ticker and quarter, then saves the raw transcript locally:

# API call -fetches the full earnings call transcript as JSON:
curl "https://financialmodelingprep.com/stable/earning-call-transcript
    ?symbol=NVDA&quarter=1&year=2024&apikey=..."

Operator: Good afternoon. My name is David, and I'll be your
conference operator today. At this time, I'd like to welcome
everyone to NVIDIA's First Quarter Earnings Call...

This is a skill, not an agent. Multiple agents reuse it in parallel. The transcript-downloader feeds every downstream analysis.

Demo 1: Agent 1

/simple-transcript-analysis

The "Hello World" of investment agents. One transcript, one hardcoded screen, structured JSON output. Already saves 3-4 hours per transcript.

We give the agent a hardcoded prompt asking it to score the transcript across six dimensions:

# Prompt (hardcoded in the skill):
"Analyze this earnings transcript. Score management confidence
  (1-10), rate guidance credibility (HIGH/MEDIUM/LOW), identify
  top 3 risks with severity and trend, extract capital allocation
  signals, and score thesis alignment (1-10). Output as JSON."

# Output from NVDA Q1 2024:

"management_confidence_score": 9
"management_confidence_rationale": "Jensen spoke with
  extraordinary conviction, calling it the 'iPhone moment'."
"guidance_credibility": "HIGH"
"thesis_alignment_score": 9
"top_3_risks": [
  { "Supply chain execution", severity: "HIGH", trend: "INCREASING" },
  { "China NEV softness", severity: "MEDIUM", trend: "STABLE" },
  { "Competition from every direction", severity: "MEDIUM", trend: "INCREASING" }
]

Works. But the prompts are hardcoded. What if management wants different questions for different companies? We'll build that in our next agent.

Adoption Strategy

Mandated AI Usage Does Not Encourage Adoption

The most effective enterprise AI rollouts follow a curated discovery pattern, not top-down mandates.

  • Curated performance discovery period: let people find tools that genuinely help their specific workflow
  • Share, don't mandate: have them share what they found useful with each other
  • Define approvals needed, then come back a few weeks later
  • Collect the tools people actually used and bring to management to show organic adoption
  • Tier by seniority: different agents are relevant at each rung in the hierarchy
Security & Compliance

"Can We Use This and Be Compliant?"

Every tool in the stack can be reviewed independently. The agent architecture is platform-agnostic.

Recommended Tech Stack

  • Claude API with zero data retention
  • Self-hosted or Azure-hosted options
  • MCP servers run on-premise, inside GBL's network
  • No data leaves to third parties
  • Audit trail on every agent action

Microsoft Ecosystem Equivalents

  • Azure OpenAI Service (EU data residency)
  • Copilot Studio for agent orchestration
  • Microsoft Fabric for data layer
  • Copilot for M365 for analyst-level tasks
  • Power Automate as MCP equivalent
